MITK AI Assistant – Project Overview

Single‑page chat assistant for campus information. Backend uses Flask; frontend is HTML/CSS/JS.

How to run

  1. Install dependencies: python -m pip install -r requirements.txt
  2. Start server: python main.py
  3. Open http://localhost:5000

Frontend

  • UI files: index.html, style.css, script.js
  • Features: timestamps, auto‑growing textarea, typing indicator, suggested follow‑ups
  • Input: Enter sends the message (no Shift behavior). Send button also works.
  • Responsive: centered container with max width, mobile‑friendly paddings
  • Note: dark mode and copy buttons were removed as requested.

Backend

  • Endpoints: GET / static, POST /chat, GET /info, GET /suggestions, GET /health
  • Rate limits (if enabled): 30/min on /chat, 120/hour on /info
  • Content: editable JSON in data/college_info.json and data/synonyms.json

Editing knowledge

Update answers in data/college_info.json, and synonyms/keywords in data/synonyms.json. Restart server to reload.

Keyboard and input

Press Enter to send. The textarea auto‑expands up to 160px.

Troubleshooting

  • “Error connecting to server”: ensure the server is running (python main.py). Then visit /health to check it returns {"status":"ok"}.
  • Open the app via http://localhost:5000, not by double‑clicking the HTML file.
  • If you see 429 (Too Many Requests) in Network tab, wait ~60 seconds and retry.
  • Hard refresh the browser (Ctrl+F5) after code changes.
